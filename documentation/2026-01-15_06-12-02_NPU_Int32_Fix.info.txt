Task: Fix NPU Driver Crash (ZE_RESULT_ERROR_UNKNOWN) by Enforcing Int32 Precision

Timestamp: 2026-01-15_06-12-02

Summary of Changes:
Modified the model baking process (`src/python/bake_model.py`) to explicitly downgrade the precision of model input tensors from `Int64` (PyTorch default) to `Int32`. This addresses the Intel Level Zero driver error `0x7ffffffe` (ZE_RESULT_ERROR_UNKNOWN), which often occurs when NPU hardware receives 64-bit indices it cannot natively process.

Detailed Implementation Notes:
1.  **`src/python/bake_model.py`:**
    *   Added logic in "Stage 2" (OpenVINO Core Reshaping) to iterate over all model inputs (`input_node` in `ov_model.inputs`).
    *   Check: `if input_node.get_element_type() == ov.Type.i64:`.
    *   Action: `input_node.get_node().set_element_type(ov.Type.i32)` and `set_output_type(0, ov.Type.i32, partial_shape)`.
    *   This forces the parameter nodes in the OpenVINO graph to declare themselves as 32-bit integers, ensuring the subsequent compilation for NPU generates a compatible blob.
    *   Updated verification logging to print both `Shape` and `Type` for confirmation.

Fixes & Caveats:
*   **Fix:** Resolves the critical `ZE_RESULT_ERROR_UNKNOWN` crash on Intel Core Ultra NPU during inference warm-up and execution.
*   **Caveat:** This requires re-baking the model. Old `.xml/.bin` files generated by previous versions of the script will still cause crashes. The user must run `python src/python/bake_model.py` again.
*   **Hardware Constraint:** Intel NPU architectures generally prefer or require 32-bit indices for embedding lookups and gather operations to optimize memory bandwidth.

Usage Instructions:
To apply this fix, you must re-generate the model binary:
```bash
python src/python/bake_model.py --model_id "Intel/neural-chat-7b-v3-1" --output_dir ./models/neuralchat_int4 --verbose
```
Then run the supervisor as normal:
```bash
python src/python/supervisor.py --model_xml ./models/neuralchat_int4
```

Technical Metrics:
*   **Target Precision:** Int32 for `input_ids`, `attention_mask`, `position_ids`.
*   **Previous State:** Int64 (Crash on NPU).
*   **New State:** Int32 (Stable on NPU).
